# 要件定義書 - Kindle Checker

## 1. プロジェクト概要

Amazon欲しいものリストに登録されたKindle本のポイント還元率・割引率をチェックし、リスト上にインライン表示するChrome拡張機能。

---

## 2. 用語定義

| 用語 | 説明 |
|---|---|
| 欲しいものリストページ | `amazon.co.jp/hz/wishlist/ls/...` または `amazon.co.jp/wishlist/...` 形式のURL |
| ポイント還元率 | 商品価格に対するAmazonポイント付与割合（%） |
| ポイント還元額 | 付与されるAmazonポイント数（pt） |
| 割引率 | 定価（参考価格）に対する値引き割合（%） |
| 割引額 | 定価から現在価格を引いた金額（円） |
| Kindle本 | ASIN が Kindle 電子書籍フォーマットの商品 |

---

## 3. 対象スコープ

- **対象ページ**: Amazon欲しいものリストページのみ（`amazon.co.jp`）
- **対象商品**: Kindle本のみ（紙の本・その他商品はスキップ）
- **対象数量**: リスト内全件（ページネーション含む全ページ）

---

## 4. 機能要件

### FR-01 ページ自動検出・実行

- **説明**: Amazon欲しいものリストページを開いた際に自動的に処理を開始する
- **トリガー**: ページロード完了時（`DOMContentLoaded` or `load` イベント）
- **優先度**: Must

### FR-02 Kindle本フィルタリング

- **説明**: 欲しいものリスト内の商品からKindle本のみを抽出する
- **判定方法**: 商品タイトルまたはフォーマット表記に以下のいずれかが含まれる商品を対象とする
  - "Kindle版"
  - "Kindle Unlimited"
  - その他 "Kindle" を含む電子書籍フォーマット表記
- **優先度**: Must

### FR-03 商品ページスクレイピング

- **説明**: 対象Kindle本の商品詳細ページにアクセスし、価格情報を取得する
- **取得方法**: 逐次取得（1件ずつ順番にfetch）
- **取得間隔**: 1リクエストあたり最低500ms以上のインターバルを設ける（Amazon Rate Limit対策）
- **取得情報**:
  - 現在価格（円）
  - 定価・参考価格（円）
  - ポイント還元額（pt）
  - ポイント還元率（%）
- **優先度**: Must

### FR-04 情報計算・補完

- **説明**: スクレイピングで取得した情報から不足分を計算する
- **計算ルール**:
  - 割引額 = 定価 − 現在価格
  - 割引率 = (割引額 ÷ 定価) × 100（小数点第1位まで）
  - ポイント還元率がページに明示されていない場合 = (ポイント還元額 ÷ 現在価格) × 100（小数点第1位まで）
- **前提**:
  - 定価が存在しない場合は割引率・割引額を非表示とする
  - ポイント情報がDOM上に存在しない場合は `0pt / 0%` として表示する
- **優先度**: Must

### FR-05 インライン表示

- **説明**: 欲しいものリスト上の各Kindle本のタイトル直下にバッジ形式で情報を表示する
- **表示内容**:
  - ポイント還元率（例: `🪙 12%還元`、ポイントなし時: `🪙 0%還元`）
  - ポイント還元額（例: `+150pt`、ポイントなし時: `+0pt`）
  - 割引率（例: `🏷️ 30%OFF`）
  - 割引額（例: `−450円`）
- **表示条件**: 取得成功時のみ表示（エラー時は別途表示、→FR-06）
- **優先度**: Must

### FR-06 エラーハンドリング

- **説明**: 個別商品の情報取得に失敗した場合、該当商品にエラー表示を行い、他の商品の処理は継続する
- **エラー表示**: タイトル下に「取得失敗」ラベルを表示（例: `⚠️ 情報取得失敗`）
- **継続処理**: エラーが発生した商品をスキップして次の商品へ進む
- **優先度**: Must

### FR-08 ページネーション対応

- **説明**: 欲しいものリストが複数ページに渡る場合も全件を対象とする
- **取得戦略**:
  1. **優先**: ページロード時にリスト全ページを先読みしてASINを収集し、まとめてスクレイピング開始
  2. **代替**: 先読みが技術的に困難な場合は、各ページが表示された時点でそのページの商品のスクレイピングを開始する
- **優先度**: Must

### FR-07 処理進捗表示

- **説明**: スクレイピング中はユーザに処理状況を通知する
- **表示内容**: ページ上部またはリスト上部に「チェック中... (n/N件)」を表示
- **完了時**: 「チェック完了 (N件中M件取得成功)」を表示
- **優先度**: Should

---

## 5. 非機能要件

### NFR-01 パフォーマンス

- 1件あたりの取得時間目標: 2秒以内（ネットワーク状況に依存）
- リクエスト間隔: 最低500ms（Amazon規約・レート制限対策）

### NFR-02 安定性

- 単一商品のエラーで全体処理が停止しないこと（FR-06参照）
- DOMの変更に対してある程度耐性を持つ実装とする（セレクタの変更に備え、複数候補を用意）

### NFR-03 セキュリティ

- 取得したデータはローカルメモリのみに保持し、外部送信は行わない
- `content_security_policy` に従い、インラインスクリプトの挿入は行わない

### NFR-04 互換性

- 動作対象: Google Chrome 最新版
- 対象サイト: `amazon.co.jp` のみ

### NFR-05 ユーザビリティ

- 拡張機能の特別な設定なしに機能すること（インストールするだけで動作）
- 既存のAmazonページのレイアウト・UXを大きく損なわないこと

---

## 6. 対象外（Out of Scope）

- キャッシュ機能（毎回最新データを取得する）
- 並列スクレイピング
- amazon.com など日本語以外のAmazonサイト
- 紙の本・その他のKindle以外商品
- 通知機能（価格変動アラート等）
- ユーザ設定UI

## 8. 確定済み仕様（旧・未決定事項）

| 項目 | 決定内容 |
|---|---|
| Kindleフィルタ精度 | "Kindle Unlimited" 等、"Kindle" を含む全電子書籍フォーマット表記を対象とする |
| ポイント情報の非表示時 | ポイントなし商品は `0pt / 0%` として表示する |
| ページネーション対応 | 可能であればページロード時に全ページのASINを先読み取得。先読みが困難な場合は各ページ表示時に取得開始 |
| 定価なし商品の扱い | 割引率・割引額は非表示とする（FR-04前提条件で対応済み） |
